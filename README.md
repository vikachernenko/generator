# Проект №2469

## Генератор нагрузочных сценариев для исследований производительности PostgreSQL
![фото](architecture_g.pdf)

Инструмент для генерации воспроизводимых нагрузочных сценариев, моделирующих типовые и проблемные режимы работы PostgreSQL, с целью проведения исследований производительности и выявления причин деградации выполнения запросов.

#### Требования: Docker

## Установка и запуск

1. В терминали используйте команду cd, чтобы перейти в каталог, куда вы хотите сохранить репозиторий.

2. Клонируйте репозиторий:
   `git clone https://git.miem.hse.ru/2469/pg_loader.git`

3. Разверните инфраструктуру: `docker compose up --build`

## Использование

1. Из контейнера переходим в прометеус `http://localhost:9090/query` -> `Status` -> `Target health`. Если там endpoint - up, то все хорошо и наш прометеус может забирать данные у экспортера.

2. Смотрим `http://localhost:9432/metrics`, если там много единичек, то все хорошо и наш pg_exporter заходит в бд (в дальнейшем распиши тут все подробнее)

3. Из контейнера переходим в графану, пороль и логин там уже написаны по умолчанию, нужно лишь нажать `Log in` -> `Skip`

Мы зашли в профиль графаны, далее нажимаем `Dashboards` -> `PostgreSQL Monitoring Dashboard` и вот наш дашборд
